QA Workflow

My process from requirements to regression testing.

1) Understand requirements Scope • Acceptance criteria
  • Read user story and acceptance criteria
  • Identify edge cases and risks
  • Prepare test data and scenarios
2) Create test cases Steps • Expected results
  • Write positive and negative cases
  • Include boundary values
  • Link to requirements
3) Execute + report bugs Evidence • Priority
  • Reproduce consistently
  • Attach screenshot / logs
  • Include clear steps + actual vs expected
4) Regression testing Verify fixes • Prevent re-break
  • Re-test the bug fix
  • Run regression suite
  • Update test cases if needed

Automation Checklist

Before adding a test to suite, I check:

  • Stable selectors (data-testid)
  • Clear assertions
  • Reliable waits (no hard sleep)
  • Independent tests (no order dependency)
  • Good naming + readable steps

Bug Report Template

Simple structure for clear reporting.

Title: Short, clear issue

Steps: 1… 2… 3…

Expected: What should happen

Actual: What happened

Evidence: Screenshot / video / logs
